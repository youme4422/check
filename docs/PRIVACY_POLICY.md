# check Privacy Policy

## 한국어

`check`는 사용자의 체크인 기록, 긴급 연락처, 앱 설정 데이터를 기기 로컬 저장소에만 저장합니다.

### 수집하는 정보

- 체크인 시각
- 체크인 이력
- 긴급 연락처 정보(이름, 휴대폰 번호, 이메일)
- 앱 설정(체크인 주기, 알림 여부, 언어)

### 수집 방식

- 사용자가 앱 화면에서 직접 입력한 정보만 저장합니다.
- 서버 전송, 계정 동기화, 광고 추적, 분석 수집을 하지 않습니다.

### 정보 이용 목적

- 체크인 상태 표시
- 체크인 이력 조회
- 긴급 연락처 빠른 연결
- 알림 예약
- 언어 및 앱 환경설정 유지

### 제3자 제공

- 사용자의 데이터를 개발자 서버나 제3자 서버로 전송하지 않습니다.
- 사용자가 직접 실행한 `sms:` 또는 `mailto:` 링크는 기기의 문자/이메일 앱으로 전달될 수 있습니다.

### 보관 및 삭제

- 데이터는 사용자의 기기에만 저장됩니다.
- 사용자는 앱 내 데이터 초기화 기능으로 언제든지 삭제할 수 있습니다.

### 아동 정보

- 별도의 아동 대상 서비스가 아니며, 연령 확인이나 계정 가입 기능이 없습니다.

## English

`check` stores check-in history, emergency contacts, and app settings only in local device storage.

### Data collected

- Check-in timestamps
- Check-in history
- Emergency contact information (name, mobile number, email)
- App settings (interval, notifications, language)

### How data is collected

- Only information entered directly by the user in the app is stored.
- The app does not use server sync, analytics tracking, or advertising tracking.

### Purpose of use

- Show check-in status
- Display history
- Open quick emergency contact actions
- Schedule reminders
- Preserve language and app preferences

### Sharing

- User data is not sent to developer servers or third-party servers.
- `sms:` and `mailto:` actions initiated by the user may pass data to the device messaging or email app.

### Retention and deletion

- Data remains only on the user device.
- Users can delete all stored data at any time with the in-app reset feature.
